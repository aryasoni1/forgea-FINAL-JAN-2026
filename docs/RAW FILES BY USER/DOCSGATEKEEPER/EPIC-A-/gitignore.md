You can tell Git which files to ignore with a .gitignore file or other ignore mechanisms.

Quick steps to create and use a .gitignore in a repository

In your repository root create the file:
touch .gitignore
Add patterns (one per line) using the same pattern rules as .gitignore (see Git docs PATTERN FORMAT).
Commit the .gitignore so others get the same rules:
git add .gitignore
git commit -m "Add .gitignore"
If a file is already tracked, stop tracking it before ignoring:

Untrack the file but keep it locally:
git rm --cached FILENAME
git commit -m "Stop tracking FILENAME"
Global and local-only ignores

Ignore files across all repositories on your computer: add patterns to ~/.config/git/ignore.
Ignore files only locally (not committed): edit .git/info/exclude inside the repository.
Templates and automation

Use GitHub's collection of recommended templates in the github/gitignore repo for OSs, languages, and frameworks: https://github.com/github/gitignore
You can fetch .gitignore templates via the REST API: Get a gitignore template and Get all gitignore templates.

About gitignore
When you create a new repository on GitHub via the API, you can specify a .gitignore template to apply to the repository upon creation. You can use the REST API to get .gitignore templates from the GitHub .gitignore repository.
